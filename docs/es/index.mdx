## Características

<AccordionGroup>
  <Accordion icon="check" title="Manejo de Errores Tipado">
    No más bloques `try-catch`. Obtén uniones discriminadas con soporte completo de TypeScript.
    ```typescript
    const result = await runner.run(() => apiCall());
    if (result.ok) {
      // TypeScript sabe que data está disponible
      console.log(result.data);
    }
    ```
  </Accordion>

  <Accordion icon="arrows-rotate" title="Lógica de Reintentos Inteligente">
    Backoff exponencial integrado con jitter para prevenir el problema de thundering herd.
    ```typescript
    await runner.run(() => unstableApi(), {
      retries: 3,
      retryDelay: (attempt) => Math.pow(2, attempt) * 1000,
      jitter: true
    });
    ```
  </Accordion>

  <Accordion icon="layer-group" title="Control de Concurrencia">
    Ejecuta múltiples operaciones con paralelismo controlado.
    ```typescript
    await runner.all(tasks, { 
      concurrency: 5,
      mode: 'fail-fast' 
    });
    ```
  </Accordion>

  <Accordion icon="ban" title="Manejo de Abort">
    Soporte de primera clase para `AbortController` y cancelación elegante.
    ```typescript
    await runner.run(() => fetch('/api', { signal }), {
      ignoreAbort: true
    });
    ```
  </Accordion>
</AccordionGroup>

## Ejemplo Rápido

```typescript
import { createRunner } from 'runtry';

const runner = createRunner();

const result = await runner.run(
  () => fetch('https://api.example.com/users'),
  {
    retries: 3,
    retryDelay: 1000,
    onError: (error) => console.error(error)
  }
);

if (result.ok) {
  const users = await result.data.json();
  console.log(users);
}
```

<Note>
  **Consejo Pro:** Usa `createRunner()` una vez y reutilízalo en tu aplicación para un manejo de errores consistente.
</Note>
